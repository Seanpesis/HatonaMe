# תיקון בעיית "אירוע לא נמצא"

## הבעיה:
האירועים שנוצרו לא מופיעים או נמחקים.

## הסיבה:
המסד נתונים SQLite נמצא **מקומית** על המחשב שלך. כשהשרת לא רץ או רץ במקום אחר (Railway), הוא לא רואה את הנתונים.

---

## פתרונות:

### פתרון 1: השרת רץ מקומית (לפיתוח)

אם אתה מריץ את השרת על המחשב שלך:

1. **ודא שהשרת רץ:**
   ```bash
   npm run dev
   ```

2. **ודא שהמסד נתונים קיים:**
   - קובץ `wedding_planner.db` צריך להיות בתיקיית הפרויקט
   - אם אין - השרת יוצר אותו אוטומטית

3. **הנתונים נשמרים מקומית:**
   - כל האירועים, מוזמנים, וכו' נשמרים בקובץ `wedding_planner.db`
   - זה עובד רק כשהשרת רץ על המחשב שלך

---

### פתרון 2: השרת על Railway (ל-Production)

אם העלית את השרת ל-Railway:

#### הבעיה:
- Railway יוצר מסד נתונים **חדש וריק**
- הנתונים המקומיים לא מועברים אוטומטית

#### פתרון - העברת נתונים:

**אפשרות A: העלה את קובץ המסד נתונים**

1. **גבה את המסד נתונים המקומי:**
   - העתק את `wedding_planner.db` למקום בטוח

2. **העלה ל-Railway:**
   - Railway → Project → Volumes
   - צור Volume חדש
   - העלה את `wedding_planner.db` לשם
   - עדכן את הנתיב ב-`server/database/db.js`

**אפשרות B: ייצא/ייבא נתונים**

1. **ייצא את הנתונים מהמסד המקומי:**
   ```bash
   # יצירת קובץ SQL
   sqlite3 wedding_planner.db .dump > backup.sql
   ```

2. **ייבא ל-Railway:**
   - העלה את `backup.sql` ל-Railway
   - הרץ את ה-SQL על המסד החדש

**אפשרות C: צור מחדש (אם אין הרבה נתונים)**
- פשוט צור את האירועים מחדש ב-Railway

---

### פתרון 3: מסד נתונים חיצוני (מומלץ ל-Production)

SQLite לא מומלץ ל-Production. עדיף PostgreSQL:

1. **Railway → New → Database → PostgreSQL**
2. **קבל את ה-Connection String**
3. **עדכן את הקוד:**
   - שנה מ-SQLite ל-PostgreSQL
   - עדכן את ה-Connection String

---

## בדיקה:

### בדוק שהשרת רץ:
פתח בדפדפן:
```
http://localhost:5000/api/events
```
או (אם על Railway):
```
https://your-project.up.railway.app/api/events
```

אמור לראות רשימת אירועים ב-JSON.

### בדוק את המסד נתונים:
```bash
# אם יש לך sqlite3 מותקן
sqlite3 wedding_planner.db "SELECT * FROM events;"
```
